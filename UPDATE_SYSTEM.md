# نظام التحديث الآمن

لا يحتوي المشروع على باب خلفي أو ملف ينفذ كودًا عن بُعد. التحديث يتم عبر GitHub، وهو المسار الآمن والقابل للتدقيق والرجوع.

## بعد الرفع الأول

1. اربط المستودع بمشروع Vercel.
2. اجعل فرع الإنتاج `main`.
3. أي Commit جديد إلى `main` يشغّل فحص GitHub Actions.
4. Vercel يلتقط التغيير وينشر نسخة جديدة تلقائيًا.

لا تحتاج بعد ذلك إلى رفع ملف ZIP كامل. يكفي تعديل الملفات المطلوبة ثم Push.

## أمر النشر الآمن

```bash
npm run release -- "وصف التحديث"
```

يقوم الأمر بالترتيب التالي:

1. يتأكد أن المشروع داخل Git.
2. ينشئ Git Tag احتياطيًا قبل التعديل.
3. يشغّل الاختبارات المحددة في `update.config.json`.
4. ينشئ Commit.
5. يرفع التحديث والنسخة الاحتياطية إلى GitHub.
6. يبدأ Vercel النشر تلقائيًا عند تفعيل Git Integration.

## الرجوع

اعرض النسخ الاحتياطية:

```bash
git tag --list "backup-*" --sort=-creatordate
```

ثم ارجع بطريقة تحفظ التاريخ ولا تستخدم Force Push:

```bash
npm run rollback -- backup-20260723T120000Z
```

## التحديث من ChatGPT

بعد ربط مستودع GitHub بحسابك، يمكن تنفيذ التعديلات مباشرة في المستودع. لا تمنح أي شخص Token داخل الملفات، ولا ترفع `.env` أو مفاتيح Vercel أو OpenAI إلى GitHub.

## حدود النظام

- تغييرات الكود تحتاج Commit جديدًا، لكنها لا تحتاج رفع ZIP من جديد.
- ربط Vercel بـGitHub مطلوب للنشر التلقائي.
- الخلفية الحالية تستخدم AppDeploy SDK؛ نقلها لتعمل داخل Vercel نفسه يحتاج ترحيل المصادقة وقاعدة البيانات والتخزين والذكاء الاصطناعي.
