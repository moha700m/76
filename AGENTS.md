## 2. المنتج — ما تحتاج معرفته قبل أي سطر كود

استوديو عربي RTL يشغّل **9 وكلاء AI متخصصين** (موجز، بحث، استراتيجية منتج،
ذاكرة علامة، UX، واجهة، كفاءة، جودة، نقد منافس) في 3 موجات، ثم يدمج المنسق
النتائج ويولّد معاينة موقع كاملة للعميل.

فوقه طبقة نمو: طابور محتوى في `content/pending-uploads/`، نشر YouTube/TikTok/
Instagram بوضع `dry-run` افتراضياً، تقارير أسبوعية، Lighthouse، نسخ احتياطي.

**اقرأ قبل أي تعديل:** `README.md` · `ARCHITECTURE.md` · `SETUP.md` ·
`DEPLOYMENT.md` · `UPDATE_SYSTEM.md` · `HYBRID_AI_ROUTING.md` · `backlog.md`

## 3. ⚠️ قيود هذا الريبو تحديداً (احفظها)

هذه أهم فقرة في الملف. تجاهلها = كود مكسور في الإنتاج.

- **الخلفية تعمل على AppDeploy SDK، لا على Vercel.** الوحدتان
  `@appdeploy/client` و `@appdeploy/sdk` غير موجودتين في CI.
  → **لا تستطيع تشغيل `npm run dev` ولا اختبار `backend/` فعلياً.**
- تحققك الوحيد المتاح: `npm run content:validate` ثم `npm run check`
  (= typecheck + unit tests + build). **لا ترفع شيئاً قبل أن ينجحا.**
- Node **22** (انظر `engines`). التنصيب: `npm install --ignore-scripts`.
- البناء على Vercel يستبدل `@appdeploy/client` بـ
  `src/platform/appdeploy-client-vercel.ts` عبر alias في `vite.config.ts`.
  أي تغيير في واجهة الـ client لازم ينعكس في هذا الملف أيضاً.
- `SHA256SUMS.txt` و `SOURCE-SNAPSHOT.json` و `SOURCE-FILES.txt` لقطات تصدير
  قديمة. **لا تحدّثها ولا تعتمد عليها.**
- `SOCIAL_PUBLISH_MODE` يبقى `dry-run`. لا تحوّله لـ `live` أبداً.
- النشر يتم عبر `npm run release`، والرجوع عبر `npm run rollback`
  (راجع `UPDATE_SYSTEM.md`). لا تستخدم force push مطلقاً.

## 4. دورة العمل اليومية

1. **استوعب.** اقرأ `.agent/STATE.md` ثم آخر 10 مدخلات في `.agent/JOURNAL.md`
   ثم `backlog.md`. اعرف وين وقفت أمس.
2. **راقب.** الـ Issues المفتوحة (خصوصاً `🚨 فشل تشغيل` و `[P0]`)،
   والـ PRs المعلقة، وآخر تقرير في `reports/weekly/`.
3. **اختر مهمة واحدة** من `backlog.md` حسب أولويتها المعلّمة (🔴 ثم 🟡 ثم 🟢)
   ومع احترام سلّم القسم 5. **مهمة واحدة فقط.** مهمة منجزة > ثلاث مهام نص شغل.
4. **نفّذ.** اكتب الكود + اختبار في `tests/unit/`. شغّل:
   ```
   npm install --ignore-scripts && npm run content:validate && npm run check
   ```
   إذا فشل — أصلحه أو ارجع للخطوة 3 بمهمة أصغر. لا ترفع كود أحمر.
5. **ارفع.** branch باسم `auto/YYYY-MM-DD-وصف-قصير`، وافتح PR بعنوان عربي
   يبدأ برمز المهمة (مثال: `SEC-002: تدوير الرموز آلياً`)، ووصفه يحوي:
   ما تغيّر، لماذا، كيف يُختبر، وما لم تستطع اختباره بسبب قيود القسم 3.
6. **حدّث backlog.** اشطب المنجز `[x]`، و**أضف مهمتين جديدتين** التزاماً
   بقاعدة المشروع: يبقى 20 مهمة مفتوحة على الأقل.
7. **وثّق.** مدخل جديد في `.agent/JOURNAL.md` + تحديث `.agent/STATE.md`.
   PR بدون تحديث اليوميات = مهمة غير مكتملة.

## 5. سلّم الأولويات

```
P0  بناء فاشل / صفحة بيضاء / 5xx في الإنتاج        ← أوقف كل شي وأصلحه
P1  ثغرة أمنية أو مفتاح مسرّب                       ← أصلح + صعّد فوراً
P2  خلل يمنع مستخدم من إكمال رحلة أساسية
P3  مهام 🔴 في backlog.md
P4  أداء / Core Web Vitals (هدف 90+، انظر .lighthouserc.json)
P5  مهام 🟡 — ميزات وتجربة و SEO
P6  مهام 🟢 — تحسينات، محتوى، refactor
```

**لا تلمس P5 وما بعدها وفي المشروع شيء عالق في P0–P3.**

## 6. متى تضيف ميزة بدون سؤال

مسموح إذا تحققت الأربعة:

1. تخدم حالة استخدام واضحة (اكتبها بجملة في الـ PR).
2. أقل من 600 سطر تغيير.
3. لا تحتاج مفتاح API جديد ولا خدمة مدفوعة جديدة ولا رفع حصة.
4. `npm run check` أخضر ولا تكسر شيئاً موجوداً.

سقط شرط؟ → أضف المهمة في `backlog.md`، وافتح Issue بعنوان `[قرار مطلوب]`
فيه الخيارات وتوصيتك، ثم **انتقل لمهمة أخرى**. لا تنتظر رداً.

## 7. الخطوط الحمراء

- ❌ تعديل `.github/workflows/` أو `AGENTS.md` أو `update.config.json` أو `package.json` أو `package-lock.json`.
- ❌ لمس `.env*` أو أي secret أو متغير Vercel.
- ❌ تحويل `SOCIAL_PUBLISH_MODE` إلى `live`، أو نشر أي محتوى فعلي.
- ❌ اعتماد ملف محتوى في `content/pending-uploads/` — الاعتماد قرار بشري.
- ❌ أي شيء يخص الدفع أو الأسعار أو الفوترة.
- ❌ حذف اختبار لتخضير CI. الاختبار الأحمر يعني الكود خطأ.
- ❌ `git push --force` أو تعديل تاريخ Git.
- ❌ إضافة dependency ثقيلة بدون سبب مكتوب في الـ PR.
- ❌ أكثر من 800 سطر في PR واحد.
- ❌ ترحيل الخلفية من AppDeploy إلى Vercel — قرار معماري كبير، صعّده.

## 8. متى تُصعّد (هذه الحالات فقط)

بصيغة: **المشكلة → الخيارات → توصيتي → ما أحتاجه منك**

1. الإنتاج واقع ولم تنجح بعد محاولتين.
2. ثغرة أمنية أو مفتاح مسرّب.
3. قرار يكلف فلوس أو يحتاج اشتراك/حساب جديد (مثل BILL-001).
4. قرار معماري كبير (ترحيل الخلفية، تغيير مزوّد AI).
5. قرار قانوني أو خصوصية بيانات، أو أي شيء يخص مراجعة TikTok.
6. مرّت 5 أيام بدون مهمة تستحق التنفيذ.

**ما عدا ذلك: قرّر ونفّذ.** متردد بين خيارين متقاربين؟ اختر الأبسط
والأسهل في التراجع عنه.
